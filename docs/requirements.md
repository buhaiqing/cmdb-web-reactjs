# 运维部 CMDB 系统需求规格说明书

## 1. 项目概述

### 1.1 项目背景
配置管理数据库（Configuration Management Database, CMDB）是运维部的核心基础设施，用于统一管理 IT 基础设施、应用服务、云资源等全部配置项。本系统旨在提供一个集中化的资源配置管理平台，支持自动发现、变更追踪、关系映射和审计合规等功能。

### 1.2 项目目标
- 建立统一的配置项管理数据库，覆盖 IT 基础设施、应用服务、云资源
- 实现与 K8s、云平台（阿里云/AWS）、监控系统（Prometheus/Grafana）的自动集成
- 提供配置项全生命周期管理，包括创建、变更、审计、版本控制
- 支持 RBAC 权限控制，确保数据安全
- 提供 UI 自动化测试友好的前端界面

### 1.3 技术栈
- **前端**: Vue 3 + TypeScript + Vite + Pinia + Element Plus
- **后端**: Python FastAPI + SQLAlchemy + PostgreSQL
- **集成**: K8s API、阿里云 API、Prometheus API
- **测试**: Playwright (UI 自动化) + Pytest (后端测试)

## 2. 功能需求

### 2.1 配置项（CI）管理

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P0 | CI 创建 | 支持创建服务器、网络设备、数据库、中间件、应用程序、容器、K8s 资源、云资源等配置项 | 用户可以成功创建各类 CI，必填字段验证通过 |
| P0 | CI 查看 | 支持列表视图和详情视图，显示 CI 的完整信息 | 列表支持分页、排序，详情页显示所有字段和关联关系 |
| P0 | CI 编辑 | 支持修改 CI 的属性信息 | 修改后数据正确保存，变更被记录到审计日志 |
| P0 | CI 删除 | 支持删除 CI（软删除） | 删除后数据标记为已删除，可通过回收站恢复 |
| P1 | 批量操作 | 支持批量导入、导出、删除 CI | 支持 Excel/CSV 格式，批量操作有事务保证 |
| P1 | 搜索过滤 | 支持多条件搜索和过滤 | 支持模糊搜索、多字段组合过滤、保存搜索条件 |
| P2 | 标签管理 | 支持为 CI 添加和管理标签 | 标签可自定义，支持按标签筛选 |

### 2.2 关系管理

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P0 | 关系创建 | 支持创建 CI 之间的依赖、连接、运行等关系 | 关系类型可选，关系双向可见 |
| P0 | 关系查看 | 支持图形化展示 CI 关系图 | 关系图清晰展示依赖关系，支持缩放和拖拽 |
| P1 | 关系删除 | 支持删除 CI 关系 | 删除关系不影响 CI 本身 |
| P1 | 影响分析 | 分析 CI 变更的影响范围 | 显示受影响的上下游 CI 列表 |

### 2.3 自动发现

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P0 | K8s 集成 | 从 Kubernetes 集群自动同步 Pod、Deployment、Service 等资源 | 支持多集群，定时同步，增量更新 |
| P1 | 云平台集成 | 对接阿里云/AWS API 自动发现云资源 | 支持 ECS、RDS、SLB、Redis 等资源类型 |
| P1 | 监控集成 | 与 Prometheus/Grafana 集成展示监控指标 | 在 CI 详情页显示关键性能指标 |
| P2 | 自动发现任务 | 配置定时自动发现任务 | 支持 cron 表达式，任务可手动触发 |

### 2.4 变更管理

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P0 | 变更记录 | 自动记录所有 CI 的变更历史 | 记录变更时间、操作人、变更内容 |
| P0 | 变更审批 | 支持变更审批流程 | 可配置审批人，审批通过后才执行变更 |
| P1 | 变更对比 | 对比 CI 的不同版本 | 高亮显示差异字段 |
| P2 | 变更通知 | 变更发生时发送通知 | 支持 webhook、邮件、钉钉等通知方式 |

### 2.5 版本管理

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P1 | 版本快照 | 支持创建 CI 版本快照 | 快照包含完整 CI 数据 |
| P1 | 版本恢复 | 支持恢复到历史版本 | 恢复后数据正确还原，生成新的变更记录 |
| P2 | 版本对比 | 对比不同版本之间的差异 | 可视化显示字段变化 |

### 2.6 报表与统计

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P1 | 资源统计 | 统计各类 CI 的数量、分布 | 支持图表展示，数据实时 |
| P1 | 合规报表 | 生成合规性检查报表 | 可导出 PDF/Excel 格式 |
| P2 | 趋势分析 | 分析 CI 变化趋势 | 支持时间范围选择，图表展示 |
| P2 | 仪表盘 | 自定义仪表盘 | 支持拖拽布局，组件可配置 |

### 2.7 权限管理

| 优先级 | 功能 | 描述 | 验收标准 |
|--------|------|------|----------|
| P0 | 用户管理 | 用户 CRUD、角色分配 | 支持批量导入用户 |
| P0 | 角色管理 | 角色 CRUD、权限配置 | 支持自定义角色 |
| P0 | 权限控制 | 基于 RBAC 的权限控制 | 未授权访问被阻止 |
| P1 | 审计日志 | 记录所有用户操作日志 | 日志不可篡改，支持审计查询 |

## 3. 非功能需求

### 3.1 性能需求
- **响应时间**: 页面加载时间 < 3 秒，API 响应时间 < 500ms
- **并发能力**: 支持 100+ 并发用户
- **数据容量**: 支持 100,000+ 配置项存储
- **同步性能**: K8s/云资源全量同步时间 < 5 分钟

### 3.2 安全需求
- **认证**: JWT Token 认证，支持 Token 刷新
- **授权**: RBAC 角色权限控制
- **加密**: 敏感数据加密存储，HTTPS 传输
- **审计**: 所有操作记录审计日志，日志保存至少 180 天
- **限流**: API 限流保护，防止 DDoS 攻击

### 3.3 可用性需求
- **系统可用性**: 99.9% 可用性
- **数据备份**: 每日自动备份，支持数据恢复
- **容灾**: 支持多可用区部署

### 3.4 可维护性需求
- **日志**: 完善的系统日志，支持日志查询和分析
- **监控**: 系统健康监控，异常情况告警
- **文档**: 完整的 API 文档、用户手册、运维手册

### 3.5 UI 自动化测试支持
- **元素标识**: 所有关键 UI 元素具有唯一的 `data-testid` 属性
- **加载状态**: 实现稳定的页面加载状态指示器
- **反馈机制**: 交互操作有明确的成功/失败反馈
- **路由设计**: 可预测的页面路由和状态管理

## 5. 用户故事

### 5.1 配置项管理

**US-001: 创建服务器配置项**
- 作为运维工程师，我想要创建服务器配置项，以便记录服务器信息
- 验收标准：
  - 填写服务器名称、IP、CPU、内存、磁盘等必填字段
  - 选择服务器类型（物理机/虚拟机/云主机）
  - 保存后可以在列表中查看到新服务器

**US-002: 搜索配置项**
- 作为只读用户，我想要搜索配置项，以便快速找到目标资源
- 验收标准：
  - 支持按名称、IP、类型等字段搜索
  - 支持模糊匹配
  - 搜索结果支持分页显示

**US-003: 查看配置项关系图**
- 作为运维工程师，我想要查看配置项的关系图，以便了解依赖关系
- 验收标准：
  - 图形化展示 CI 之间的依赖关系
  - 支持缩放和拖拽
  - 点击节点可查看 CI 详情

### 5.2 变更管理

**US-004: 提交变更申请**
- 作为运维工程师，我想要提交变更申请，以便修改配置项
- 验收标准：
  - 填写变更原因和变更内容
  - 选择审批人
  - 提交后等待审批

**US-005: 审批变更申请**
- 作为系统管理员，我想要审批变更申请，以便控制变更风险
- 验收标准：
  - 查看变更详情和影响范围
  - 可以批准或拒绝
  - 审批结果通知申请人

**US-006: 查看变更历史**
- 作为审计员，我想要查看变更历史，以便进行合规审计
- 验收标准：
  - 按时间、操作人、CI 类型等筛选
  - 查看变更详情和对比
  - 导出变更报表

### 5.3 自动发现

**US-007: 配置 K8s 同步任务**
- 作为系统管理员，我想要配置 K8s 同步任务，以便自动同步集群资源
- 验收标准：
  - 配置 K8s 集群连接信息
  - 设置同步频率
  - 手动触发同步测试

**US-008: 查看同步结果**
- 作为运维工程师，我想要查看同步结果，以便确认资源配置正确
- 验收标准：
  - 查看同步成功的资源列表
  - 查看同步失败的资源及原因
  - 支持手动修复失败的同步
